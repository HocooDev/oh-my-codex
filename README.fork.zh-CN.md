# HocooDev oh-my-codex Fork

**语言：** [English](./README.fork.en.md) | 简体中文

这个 fork 保留了原版 `oh-my-codex` 的整体形态，但对日常使用中的几个关键环节做了调整：助手如何接收指令、长任务如何推进、安装状态如何检查，以及任务结束时如何收尾。

它仍然以 `oh-my-codex` 的方式使用，主命令仍是 `omx`。

## 这个 fork 改了什么

### 1. 角色指令的组织更清晰

原版项目中，许多角色指令以 prompt 文件的形式存在。这个 fork 将这些指令整理为可安装的 skill 文件。

实际效果是：角色行为更容易安装、刷新和检查，也更不容易因为旧 prompt 文件残留，导致助手在 setup 之后仍按旧规则工作。

### 2. 大型任务推进更少中断

面对较大的请求时，助手现在有更明确的规则来判断：什么时候继续执行，什么时候复核，什么时候必须停下来询问用户。

这并不是为了让助手不加区分地“自动执行”。它的目标是让安全、可撤回的工作顺畅推进，同时在真正需要用户判断的地方停下来。

### 3. 每个任务结束时都会做一次收尾检查

这个 fork 将收尾复核变成了任务流程的一部分。

任务结束时，助手应当说明：

- 完成了什么；
- 是否需要更新项目笔记；
- 是否需要更新项目指令；
- 是否有可复用的 skill 或规则需要调整。

这份报告会跟随对话语言输出。中文对话使用中文格式，英文对话使用英文格式，其他语言也可以自然适配。

### 4. setup 刷新更严格

`omx setup` 以及相关检查会更谨慎地处理旧的生成文件。

这能减少一类常见问题：setup 看起来成功了，但本地旧文件仍在影响助手行为。这个 fork 更倾向于干净刷新、明确提醒，并在确认安装健康前做更充分的检查。

### 5. 新增 `omx ready` 预检查

这个 fork 新增了：

```bash
omx ready
```

你可以在新安装或刷新配置后先运行它。它会检查本地环境、Codex 连接状态，以及是否能够执行一个基础命令。

这在 Windows 或一些终端环境中尤其有用，因为登录状态、shell 行为和命令可见性有时会与预期不同。

### 6. 互动提问会安全失败

有些任务需要显示一个提示，让用户回答问题。但在后台会话或非交互式终端中，这个提示可能根本不可见。

这个 fork 会避免在这种情况下无限等待。如果无法确认提示可见，提问流程会尽早停止，而不是静默卡住。

### 7. 提醒功能更容易维护

提醒和通知相关的逻辑被整理得更清楚，不再和其他命令流程混在一起。

对用户来说，结果应该很直接：隐藏失败更少，提醒更稳定，出问题时也更容易检查。

## 保持不变

- 主命令仍是 `omx`。
- 包名称仍是 `oh-my-codex`。
- 项目仍是 OpenAI Codex CLI 的辅助层。
- 上游文档链接和 MIT 许可证保持不变。

## 本地构建与检查

在仓库根目录执行：

```bash
npm install
npm run build
npm run verify:native-agents
```

除非另有说明，这个 fork 尚未作为独立 npm 包发布。
